API Dokumentasi v1

API mengikuti konvensi RESTful dan mengembalikan respons dalam format JSON. Endpoint dan response kompatibel dengan format JasaOTP, jadi web yang sudah dirancang untuk JasaOTP cukup mengganti base URL.

Autentikasi

Semua permintaan API memerlukan parameter api_key untuk autentikasi. API key bisa didapat dari panel admin web ini.

API key bisa dikirim lewat salah satu cara berikut:

  • Query parameter: ?api_key=sk_xxxxxxxx
  • Header: X-API-Key: sk_xxxxxxxx
  • Header: Authorization: Bearer sk_xxxxxxxx
⚠️ Jaga kerahasiaan API key. Jangan disimpan di kode frontend yang publik.

Base URL

Semua permintaan dibuat ke URL dasar:

https://api.happypixel.net/v1/

Flow Pemakaian

  1. Cek saldo dengan /v1/balance
  2. Cek negara/operator/layanan yang tersedia (/v1/negara, /v1/operator, /v1/layanan)
  3. Buat pesanan dengan /v1/order → dapat order_id + nomor HP
  4. Tunggu OTP dengan SSE (/v1/sms/stream) atau polling (/v1/sms)
  5. Kalau OTP tidak masuk, batalkan dengan /v1/cancel untuk refund
💡 Setelah /v1/order dipanggil, server otomatis melakukan polling ke upstream di background. Jadi kalaupun web order sempat disconnect, OTP yang masuk tetap tersimpan di database. Polling berhenti otomatis ketika OTP diterima atau upstream melaporkan order expired.

Buat Pesanan

GET/v1/order

Membuat pesanan OTP baru.

ParameterWajibTipeDeskripsi
api_keyYaStringAPI Key Anda
negaraYaIntegerID Negara (misal "6" untuk Indonesia)
layananYaStringKode layanan (misal "wa" untuk WhatsApp)
operatorYaStringNama operator (misal "any", "telkomsel")
GET /v1/order?api_key=sk_xxxxxxxx&negara=6&layanan=wa&operator=any
{
  "code": 200,
  "success": true,
  "message": "Order berhasil.",
  "data": {
    "order_id": "00000123",
    "number": "+6282272111384"
  }
}
order_id bertipe string 8 digit (sequential, dengan leading zero seperti "00000001"). Simpan untuk dipakai di endpoint berikutnya.

Periksa OTP

GET/v1/sms

Mengambil OTP untuk pesanan.

ParameterWajibTipeDeskripsi
api_keyYaStringAPI Key Anda
idYaStringorder_id dari /v1/order
GET /v1/sms?api_key=sk_xxxxxxxx&id=00000123

Saat OTP sudah masuk:

{
  "code": 200,
  "success": true,
  "message": "Berhasil mengambil OTP.",
  "data": { "otp": "123456" }
}

Saat OTP belum masuk:

{
  "code": 200,
  "success": true,
  "message": "Masih menunggu kode OTP.",
  "data": { "otp": "Menunggu" }
}
Cara membedakan OTP asli dari status pending: validasi data.otp harus angka 3-10 digit, contoh: /^\d{3,10}$/.test(data.otp). Jika cocok berarti OTP asli, jika tidak ("Menunggu", "Timeout", dll) berarti belum ada.
Jika polling, beri jeda minimal 3 detik antar request. Atau gunakan SSE untuk real-time tanpa polling.

SSE Stream real-time

GET/v1/sms/stream

Server-Sent Events. Buka 1 koneksi, terima OTP otomatis pas masuk. Lebih efisien dari polling — tidak perlu loop di sisi client.

ParameterWajibTipeDeskripsi
api_keyYaStringAPI Key Anda
idYaStringorder_id

Contoh JavaScript:

const url = '/v1/sms/stream?id=00000123&api_key=sk_xxxxxxxx';
const es = new EventSource(url);

es.addEventListener('connected', e => {
  // { order_id: "00000123", status: "pending" }
  console.log('Standby:', JSON.parse(e.data));
});

es.addEventListener('otp', e => {
  // { type: "otp", public_id: "00000123", otp: "123456" }
  const { otp } = JSON.parse(e.data);
  console.log('OTP masuk:', otp);
  es.close();
});

es.addEventListener('timeout', e => {
  console.log('Order timeout / sudah expired di upstream');
  es.close();
});

Event yang dikirim:

EventKapanPayload
connectedSaat koneksi dibuka{ order_id, status }
otpSaat OTP berhasil diterima (langsung tutup koneksi){ type, public_id, otp }
timeoutSaat upstream JasaOTP melaporkan order expired/cancelled{ type, public_id }

Server juga mengirim heartbeat berupa comment SSE (: heartbeat) tiap 15 detik untuk menjaga koneksi tetap hidup di balik proxy/CDN.

Batalkan Pesanan

GET/v1/cancel

Membatalkan pesanan yang ada dan mendapatkan pengembalian dana.

ParameterWajibTipeDeskripsi
api_keyYaStringAPI Key Anda
idYaStringorder_id
GET /v1/cancel?api_key=sk_xxxxxxxx&id=00000123
{
  "code": 200,
  "success": true,
  "message": "Pesanan berhasil dibatalkan dan saldo dikembalikan.",
  "data": {
    "order_id": "00000123",
    "refunded_amount": 3000
  }
}

Cancel hanya berlaku untuk order yang masih pending. Order yang sudah menerima OTP tidak bisa di-refund.
