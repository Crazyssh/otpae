API Documentation v1

Panduan lengkap untuk menggunakan API OTP. API ini mengikuti konvensi RESTful dan mengembalikan respons dalam format JSON.

Authentication

Semua endpoint di bawah /v1/* memerlukan API key. API key bisa dikirim lewat:

  • Query parameter: ?api_key=sk_xxxxxxxx
  • Header: X-API-Key: sk_xxxxxxxx
  • Header: Authorization: Bearer sk_xxxxxxxx
⚠️ Jaga kerahasiaan API key Anda. Jangan disimpan di kode frontend yang publik.

Base URL

https://your-domain.com/v1/

Response Format

Semua response mengikuti format yang konsisten:

{
  "code": 200,
  "success": true,
  "message": "Berhasil ...",
  "data": { ... }
}

Setelah order dibuat, server otomatis mulai polling OTP di background. Anda bisa cek lewat /v1/sms, atau langsung subscribe ke /v1/sms/stream untuk real-time.

SSE Stream real-time

GET/v1/sms/stream
ParameterWajibTipeDeskripsi
idYaStringOrder ID

Server-Sent Events untuk OTP real-time. Lebih efisien daripada polling.

const es = new EventSource('/v1/sms/stream?id=5849273011&api_key=sk_xxxxxxxx');

es.addEventListener('connected', e => console.log('Standby:', JSON.parse(e.data)));
es.addEventListener('otp', e => {
  const { otp } = JSON.parse(e.data);
  console.log('OTP masuk:', otp);
  es.close();
});
es.addEventListener('timeout', () => {
  console.log('Order timeout');
  es.close();
});

Event yang dikirim:

  • connected — koneksi terbentuk
  • otp — OTP berhasil diterima
  • timeout — OTP tidak masuk dalam waktu yang ditentukan
